DV360 is your biggest impression source but weakest converter

DV360 delivers 4.21M impressions — 34.8% of total — but converts at only 0.8% per click, the lowest across all channels. It's not a budget problem (R278K is appropriate for display). It's an audience targeting or creative problem. Recommendation: tighten audience segments, exclude low-intent placements, and A/B test creatives before scaling this channel.

Pipeline Reliability

Data you can trust — built on a medallion pipeline

The numbers above are only reliable because bad data was caught before it reached Gold. Apache Beam quarantined 6.0% of Bronze rows — not deleted them. Every quarantined row is auditable.

📥
BRONZE
Raw ingestion from 5 sources via BQ Data Transfer. All columns STRING — zero rows rejected. Mixed dates, currency variants, and missing fields land exactly as received.
523,480 rows · marketing_bronze.* · daily append
🔍
BEAM VALIDATION
Apache Beam ValidateAndTagRow DoFn routes each row: valid to Silver, bad to quarantine. 31,409 rows flagged — mostly YouTube date format issues and Ads missing currency.
94.0% pass · 6.0% quarantine · zero rows dropped
🔧
SILVER — dbt
dbt staging models type-cast, deduplicate, and normalise. Cost micros ÷ 1,000,000. Currency uppercased and trimmed. Incremental merge on unique_key prevents double-counting.
492,071 rows · 5 dbt models · 18 tests all passing
📊
GOLD — Analytics
fct_cross_channel_performance: one row per channel per day. Single source of truth for all ROAS, CTR, and conversion metrics. Full overwrite — always idempotent.
marketing_gold.* · Looker Studio · Airflow 04:00 UTC daily
💡
Why the pipeline design matters for the numbers above

Without the Beam validation step, cost micros from Google Ads would inflate spend figures by 1,000,000×. A raw cost_micros value of 338,138,553 looks like R338M — it's actually R338.14. This is the most common Google Ads data engineering mistake. The dbt Silver model normalises it before any ROAS is calculated, ensuring every number in this dashboard is correct.

YouTube has the worst DQ at 91.5% — action needed at source

YouTube's 8,172 quarantined rows are primarily driven by inconsistent date formats and missing or invalid required metadata across the Data Transfer export. This is fixable at source — YouTube Data Transfer can be configured to enforce consistent date formatting. Fixing this would recover ~8K rows and improve YouTube's ROAS attribution accuracy.
